● Here's the spec summary: clipstrike is a simple Linux CLI that converts text
  to strikethrough by inserting U+0336 after each character. It reads/writes the
   Linux clipboard by default, with stdin/stdout/file alternatives.

  Given the simplicity of the tool, here are some good language options:

  1. Bash/Shell Script

  - Pros: Zero dependencies beyond xclip/xsel for clipboard; trivial to
  distribute; natural fit for a tiny text-processing CLI
  - Cons: Unicode handling in bash can be fragile; argument parsing is clunky

  2. Python

  - Pros: Excellent Unicode support; easy argument parsing (argparse); clipboard
   via subprocess calling xclip/xsel or pyperclip; nearly universal on Linux
  - Cons: Startup overhead (~30ms); needs Python installed (though almost always
   present)

  3. Rust

  - Pros: Single static binary with no runtime deps; fast startup; good CLI
  ecosystem (clap, arboard for clipboard); strong Unicode handling
  - Cons: Heavier toolchain for a small utility; longer compile times

  4. Go

  - Pros: Single static binary; simple language for a simple tool; decent
  clipboard libraries; fast compilation
  - Cons: Binaries are relatively large for what this does

  5. C

  - Pros: Tiny binary; no runtime dependencies; fastest startup
  - Cons: Manual Unicode/UTF-8 handling is tedious and error-prone; clipboard
  interaction requires linking against X11/Wayland libs or shelling out

  My recommendation: Python or Rust. Python is the fastest path to a working
  tool with clean code. Rust is ideal if you want a self-contained binary with
  no runtime dependencies. Which would you prefer?

● The issue is that on Linux (X11), arboard sets the clipboard by becoming the
  selection owner. When the process exits, the selection is lost. The fix is to
  use arboard's SetExtLinux trait to set the clipboard in "foreground" mode
  which waits for another application to paste, or to use the
  LinuxClipboardKind::Clipboard with the clipboard manager protocol.

  Let me check what arboard offers for this.
